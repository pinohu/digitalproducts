# 01 · Market Research

Where we validate that a starving crowd exists *before* we build anything. Hormozi's hierarchy: **Starving Crowd > Offer Strength > Persuasion Skills.** Pick wrong here and nothing downstream saves the launch.

## Files in this folder

- [`starving-crowd-checklist.md`](./starving-crowd-checklist.md) — The 4-criteria filter every product must pass.
- [`niche-validation-template.md`](./niche-validation-template.md) — Fill-in-the-blank validation worksheet, one per product.
- [`competitor-analysis-template.md`](./competitor-analysis-template.md) — Map the existing landscape before deciding how to differentiate.
- `by-product/` — Completed validation work, one folder per product slug.

## Process

1. Open `niche-validation-template.md`, copy to `by-product/<slug>/validation.md`.
2. Fill it in. If any criterion fails, do not build. Either fix the positioning or kill the idea.
3. Run `competitor-analysis-template.md` for the same niche.
4. Pre-sell signal check (waitlist, landing page, or 10-buyer beta).
5. Only then move to `02-offers/` to construct the Grand Slam Offer.

## Heuristics

- If you can't find 5 competitors in the space, the market is too small *or* you haven't searched hard enough. Both are red flags.
- If competitors are charging $9 for what we plan to charge $49, we need either dramatically better positioning or a different niche.
- If the audience is on Twitter/X arguing about the problem, that's a green light. If nobody's talking about it, we're either too early or it doesn't matter enough to pay for.
