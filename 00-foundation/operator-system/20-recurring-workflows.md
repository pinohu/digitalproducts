# 20 Recurring Workflows

These are the recurring workflows Hermes and Paperclip should be able to execute repeatedly with minimal reinvention.

## 1. Board Triage

- Trigger: start of work cycle or heartbeat wake
- Inputs: open Paperclip issues, sprint docs, roadmap
- Output: prioritized next actions, truthful issue statuses

## 2. Repo Truth Sync

- Trigger: after any meaningful work
- Inputs: changed files, issue results, launch status
- Output: updated `ROADMAP.md`, sprint docs, or relevant README files

## 3. Drive Research Intake

- Trigger: new project, unclear requirement, or missing context
- Inputs: Google Drive knowledge base
- Output: distilled notes linked back to source docs

## 4. Idea Discovery

- Trigger: backlog expansion or adjacent product search
- Inputs: market research assets, trends, AppSumo portfolio, repo framework
- Output: ranked ideas with buyer/problem statements

## 5. Validation Brief Creation

- Trigger: idea passes initial filter
- Inputs: research notes, buyer pains, competitor landscape
- Output: validation brief and proof requirements

## 6. Offer Engineering

- Trigger: validated concept
- Inputs: value equation, objections, positioning notes
- Output: pricing, bonuses, guarantee, offer brief

## 7. Product Scaffold Build

- Trigger: approved offer
- Inputs: product slug, manuscript plan, asset checklist
- Output: product folder, deliverable structure, draft manuscript assets

## 8. Sales Page Drafting

- Trigger: offer and product brief ready
- Inputs: offer brief, product deliverables, proof points
- Output: sales page draft in repo

## 9. Email Workflow Drafting

- Trigger: product and sales page ready
- Inputs: launch stage, audience state, CTA target
- Output: launch emails, nurture emails, post-purchase sequence

## 10. Landing Page Deployment

- Trigger: previewable sales asset exists
- Inputs: page source, Vercel settings, environment variables
- Output: preview deployment and deployment notes

## 11. Platform Readiness Audit

- Trigger: pre-launch or pre-production review
- Inputs: domain, storefront, analytics, fulfillment, email
- Output: blocker list with owner and next step

## 12. Browser-Based SaaS Configuration

- Trigger: no usable API or incomplete API coverage
- Inputs: authenticated browser profile and SOP
- Output: completed low-risk configuration changes with logs

## 13. Social Content Planning

- Trigger: launch prep or weekly content cycle
- Inputs: offer, product, audience pains, traffic strategy
- Output: scheduled or ready-to-post content batch

## 14. Calendar and Booking Ops

- Trigger: service sales or consultation offer
- Inputs: booking rules, availability, brand, CTA pages
- Output: booking page updates and workflow notes

## 15. CRM and Lead Routing

- Trigger: new lead sources or new funnel motion
- Inputs: lead capture tools, CRM, email provider, automations
- Output: tags, pipelines, automations, and assignment logic

## 16. Workflow Automation Maintenance

- Trigger: broken automation, new integration, or repeatable manual work
- Inputs: API credentials, workflow spec, business rule
- Output: patched automation plus verification notes

## 17. Analytics Review

- Trigger: weekly or post-launch cadence
- Inputs: traffic, conversion, revenue, channel data
- Output: action-oriented review and next experiments

## 18. Customer Support Triage

- Trigger: inbound questions or issue backlog
- Inputs: customer messages, product docs, order state
- Output: resolved responses, escalations, or FAQ updates

## 19. Asset Library Expansion

- Trigger: recurring asset need appears twice
- Inputs: sales pages, bonuses, SOPs, templates
- Output: reusable shared asset in repo

## 20. Adjacent Project Advance

- Trigger: primary project blocked by external access
- Inputs: blocker list, next-best repo/project
- Output: continued momentum instead of idle time

## Workflow Rule

Each workflow should prefer this order:

1. repo truth
2. available API
3. browser automation
4. documented blocker if neither is possible
