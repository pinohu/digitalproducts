# Start Here - The Good Parts of SuiteDash

Thanks for grabbing The Good Parts of SuiteDash.

This package is designed to help you make one clean SuiteDash deployment decision fast. Do not try to use everything at once.

## Recommended order

1. Read `90-Minute-SuiteDash-Lock-In-Protocol.pdf`
2. Read `The-Good-Parts-of-SuiteDash.pdf`
3. Use `7-SuiteDash-Automation-Recipes.pdf` only after the first manual workflow is clear
4. Keep `SuiteDash-Kill-List.pdf` nearby whenever you feel yourself adding extra features

## What success looks like

By the end of your first pass, you should have:

- one workflow chosen,
- one minimum record structure,
- one intake path,
- one client-facing portal path,
- one repeatable project template,
- and one useful automation.

That is enough for version one.

## Suggested first workflow

If you are not sure where to start, use this:

client onboarding -> portal access -> project template -> file request -> status update

## Do not do this on your first pass

- do not configure multiple workflows at once
- do not turn on extra modules "just in case"
- do not build complex automations before the manual path works
- do not migrate messy historical data before your field structure is stable
