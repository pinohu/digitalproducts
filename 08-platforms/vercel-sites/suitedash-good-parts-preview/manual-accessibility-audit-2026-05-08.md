# Manual Accessibility and Usability Audit — 2026-05-08

Surface: SuiteDash Good Parts static preview landing page
URL tested locally: `http://127.0.0.1:4173/index.html`
Scope: keyboard flow, focus order, semantic structure, screen-reader clarity, mobile readability, CTA clarity, and WCAG 2.2 AA risks that automation may miss.

## Executive summary

Result: no blocking accessibility or usability issue remains on the static preview surface after this pass.

Two small remediations were made during the audit:

1. `main#main-content` now has `tabindex="-1"` so the skip link moves programmatic focus to the main landmark instead of only scrolling the page.
2. The gold accent token changed from `#b7791f` to `#9a5f16`; the previous value was a manual contrast risk for 16px bold `.stack-value` text on light cards, while the new token is at least 4.85:1 against the tested light backgrounds.

Remaining risk: human assistive-technology confirmation only. The static markup, keyboard behavior, mobile fit, and automated axe runtime checks look acceptable, but a final human pass in VoiceOver/NVDA/JAWS should confirm the reading cadence before public launch.

## Keyboard flow and focus order

Observed focus sequence:

1. `Skip to content`
2. `Review launch status`
3. `See what is included`
4. `See production blockers`

The disabled checkout state is intentionally a non-focusable `<span aria-disabled="true">Checkout pending activation</span>`, so it does not create a dead keyboard stop before the checkout is real.

Skip-link verification after remediation:

- First Tab exposes the skip link visibly at the top-left of the viewport.
- Activating it updates `location.hash` to `#main-content`.
- `document.activeElement` becomes `main#main-content`.

This satisfies the expected keyboard bypass behavior for the hero/header region.

## Semantic structure and screen-reader clarity

Positive observations:

- The page has one `h1` and a logical `h2`/`h3` hierarchy with no skipped heading levels.
- The main content is inside a `main` landmark.
- Repeated cards use `article` elements with descriptive headings.
- The hero SVG is wrapped in `aria-hidden="true"`, avoiding duplicate announcement of decorative/marketing art; its essential message is already represented in nearby text.
- CTA link text is specific enough in context: `Review launch status`, `See what is included`, and `See production blockers`.

Non-blocking note:

- The page has several unlabeled `section` elements. This is acceptable for a short static landing page because the heading hierarchy is clear, but labels can be added later if sections become independently navigable regions.

## Mobile readability and CTA clarity

Observed at 390x844:

- No horizontal overflow.
- Hero H1 fits within the viewport at 308px measured width.
- Primary CTAs stack cleanly and keep a 52px hit target.
- The checkout state clearly says `Checkout pending activation`, matching the staging truth that this is not yet a live checkout.

Launch-copy note:

- CTA clarity is appropriate for staging. Before production, the primary purchase CTA should be replaced with the real Gumroad checkout/product link and should not remain an in-page status link.

## WCAG 2.2 AA risk review

Resolved in this pass:

- Skip-link target focus robustness: fixed with `tabindex="-1"` on `main#main-content`.
- Gold text contrast risk: fixed by darkening `--gold` to `#9a5f16`.

No current blocker found for:

- Keyboard operation
- Focus visibility
- Heading order
- Landmark availability
- Mobile reflow at 390px
- Touch target size for visible CTAs
- Disabled checkout state behavior

Remaining verification before public launch:
