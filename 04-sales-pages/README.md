# 04 · Sales Pages

The sales page is where attention converts to revenue. There's a proven 12-section structure that virtually every high-converting digital product page uses (PASTOR, AIDA, PAS frameworks all collapse into the same skeleton).

## Files in this folder

- [`sales-page-structure.md`](./sales-page-structure.md) — The canonical 12-section structure, with copy guidance for each section.
- [`headline-formulas.md`](./headline-formulas.md) — 10+ proven headline formulas with examples.
- `proven-templates/` — Anonymized swipe of high-performing sales pages from the digital product space.
- `by-product/` — One file per product, the full sales page draft.

## The Process

1. Open `sales-page-structure.md` and create a new file at `by-product/<product-slug>.md`.
2. Fill in each of the 12 sections.
3. Pull from `headline-formulas.md` for the H1.
4. Pull testimonials from `/shared-assets/testimonial-bank/` once they exist.
5. Once the page is drafted, ship it — even ugly. Iterate based on conversion data. **Don't perfect; ship.**

## Conversion Targets (Industry Benchmarks)

| Metric | Cold Traffic | Warm Traffic (email list) |
|---|---|---|
| Sales page conversion | 1–3% | 5–15% |
| Founding-price launch (to list) | — | 10–20% |
| Email open rate | — | 25–40% |
| Click-through to sales page | — | 5–10% |
| Cart abandonment recovery | 5–15% | 10–20% |

If conversion is below these floors, the issue is almost always: weak headline, missing risk reversal, or pricing that doesn't match perceived value.
