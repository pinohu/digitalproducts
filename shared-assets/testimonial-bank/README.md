# Testimonial Bank

Every testimonial collected, tagged for fast retrieval when building sales pages and launch emails.

## File Pattern

One file per product: `<product-slug>.md`. Inside, all testimonials tagged with:
- Buyer name
- Buyer context (role, company size, avatar fit)
- Specific outcome / metric mentioned
- Date collected
- Permission level (full name OK / first name only / anonymous)
- Themes (e.g., "time-saved", "specific-feature", "ROI")

## Format Per Testimonial

```
### [Buyer Name] — [Date]

**Context:** [role, what they do]
**Permission:** [full / first-only / anonymous]
**Themes:** [tag1, tag2, tag3]

> [The testimonial itself, verbatim]

**Source:** [email reply / form submission / DM / etc.]
```

## When Pulling for Sales Pages

Match the testimonial to the section of the sales page:
- **Pain agitation:** testimonials describing the pain before they used the product
- **Promise / transformation:** testimonials describing the after-state
- **Specific outcomes:** testimonials with numbers / time / dollars
- **Risk reversal:** testimonials that mention how easy it was to start

A great sales page has 3–5 testimonials, each placed strategically for a specific objection.

## Always Get Permission

Before using a testimonial, confirm permission level. The testimonial request email (in `/05-email-workflows/post-purchase-sequence.md`) should be explicit about how the testimonial will be used.
