# Pre-Launch Checklist

Run this 3 days before launch. Every box must be checked. If any fails, push the launch — don't ship broken.

## Product

- [ ] Final PDF / files uploaded to Gumroad
- [ ] All bonuses uploaded and labeled clearly
- [ ] File names are clean (`the-good-parts-of-suitedash-v1.pdf`, not `final-final-FINAL-v3.pdf`)
- [ ] Total file size under any platform limits
- [ ] Test download confirms files open correctly on mobile and desktop

## Sales Page

- [ ] All 12 sections present (see `/04-sales-pages/sales-page-structure.md`)
- [ ] Headline finalized
- [ ] At least 3 testimonials placed
- [ ] Pricing section shows founding-price stack-and-strike
- [ ] Money-back guarantee visible in section 8
- [ ] FAQ has at least 7 questions
- [ ] Final CTA button works and goes to checkout
- [ ] Mobile rendering tested on actual phone (not browser dev tools)
- [ ] Page load time under 3 seconds
- [ ] No broken links (test all CTAs and external links)

## Pricing & Discount

- [ ] Founding-price discount code created
- [ ] Discount code expiration set to exact deadline (timezone confirmed)
- [ ] Public price configured to post-founding-price ($Y)
- [ ] Test purchase with discount code confirms the right price
- [ ] Test purchase without discount code confirms full price

## Email Workflows

- [ ] Post-purchase sequence (`/05-email-workflows/post-purchase-sequence.md`) live with all 6 emails
- [ ] Each email tested via 100%-discount-code test purchase
- [ ] Reply-to email goes to a monitored inbox
- [ ] Abandoned cart email customized (see `abandoned-cart.md`)

## Launch Sequence

- [ ] All 6 launch emails drafted and ready in email platform
- [ ] First email scheduled to send at 9 AM EST on Day 0
- [ ] Subsequent emails scheduled OR ready to send manually
- [ ] Waitlist segment confirmed in email platform

## Social

- [ ] Day 0 launch post drafted for primary platform
- [ ] Day 0 launch post drafted for secondary platforms
- [ ] Header image / cover art ready for each platform's dimensions
- [ ] At least 3 follow-up posts queued for Days 1–6

## Support

- [ ] Support email is monitored
- [ ] Refund process is clear (you know exactly how to refund someone in Gumroad — practice once)
- [ ] FAQ-style autoresponder considered for high volume (optional)

## Personal

- [ ] Calendar cleared for Day 0 (no meetings, no deep work blocks — be available)
- [ ] Phone notifications on for sales alerts
- [ ] You've slept. Don't launch tired.

---

## The "30 Minutes Before Launch" Final Pass

- [ ] Open the sales page in an incognito browser. Read it as if you've never seen it.
- [ ] Click the buy button. Make sure it goes to checkout.
- [ ] Verify the price displayed at checkout matches the founding price.
- [ ] Check that the discount code is active.
- [ ] Take a deep breath.

Then launch.
